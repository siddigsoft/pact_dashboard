# PACT Workflow Platform - Quick Start Cards

Role-specific quick reference guides for daily operations.

---

## Data Collector Quick Start

### Your Daily Workflow

```
1. LOGIN          → Check dashboard for new assignments
2. CLAIM          → Browse and claim available sites
3. TRAVEL         → Navigate to site location
4. START VISIT    → Click "Start Visit" when you arrive
5. COLLECT DATA   → Complete required assessments
6. COMPLETE       → Click "Complete Visit" and submit
7. CHECK WALLET   → Verify payment credited
```

### Key Pages for You

| Task | Where to Go |
|------|-------------|
| See your sites | Dashboard → My Sites |
| Find sites to claim | MMP → Claimable Sites tab |
| Start/Complete visit | Site Visit Detail page |
| Check earnings | Wallet page |
| Request withdrawal | Wallet → Request Withdrawal |
| Submit costs | Cost Submission page |

### How to Claim a Site

1. Go to **MMP** page
2. Click **Claimable Sites** tab
3. Find an available site (green "Claim" button)
4. Review site details
5. Click **Claim**
6. Confirm your claim
7. Site appears in **My Sites**

### How to Complete a Visit

1. Travel to the site location
2. Open site from **My Sites**
3. Click **Start Visit** (GPS recorded)
4. Complete your assessment
5. Take required photos
6. Click **Complete Visit**
7. Submit and confirm

### Fee Information

Your fee is calculated based on your classification:
- **Class A**: Highest rate (senior/experienced)
- **Class B**: Medium rate (intermediate)
- **Class C**: Standard rate (entry level)

Fee is automatically credited to your wallet when the visit is completed.

### Tips for Success

- Claim sites close to your location to save travel time
- Start visit only when you're physically at the site
- Complete visits the same day you start them
- Take clear, well-lit photos for documentation
- Check wallet balance after each completed visit

## Coordinator Quick Start

### Your Daily Workflow

```
1. CHECK DASHBOARD    → Review state-filtered metrics
2. MONITOR PROGRESS   → Track site visit completion
3. VERIFY SITES       → Approve completed visits
4. SUPPORT TEAM       → Assist enumerators as needed
5. REPORT             → Generate progress reports
```

### Key Pages for You

| Task | Where to Go |
|------|-------------|
| View your state's sites | Dashboard (auto-filtered) |
| Manage sites | Coordinator → Sites |
| Verify completed visits | Coordinator → Sites for Verification |
| Claim sites (if needed) | MMP → Claimable Sites |
| Track your earnings | Wallet |
| View team members | Field Team |

### Dashboard Filtering

Your dashboard automatically shows data for your assigned state:
- A badge displays "State: [Your State Name]"
- All metrics reflect only your state's data
- If no state assigned, you see all states

### How to Verify a Site Visit

1. Go to **Coordinator → Sites for Verification**
2. Review the completed visit queue
3. Click on a visit to review
4. Check:
   - GPS location accuracy
   - Photo quality
   - Data completeness
   - Visit duration
5. Click **Approve** or **Request Correction**
6. Add notes if needed

### Verification Checklist

Before approving, verify:
- [ ] Enumerator visited correct location
- [ ] GPS coordinates match site
- [ ] Required photos are clear
- [ ] Data is complete and accurate
- [ ] Visit duration is reasonable
- [ ] No obvious data quality issues

### Managing Your Team

1. Go to **Field Team** page
2. View enumerators in your state
3. Check their:
   - Current assignments
   - Workload status
   - Recent activity
4. Coordinate site distribution

### Tips for Success

- Verify sites daily to prevent backlog
- Provide constructive feedback on corrections
- Monitor team workload balance
- Escalate issues promptly
- Generate weekly progress reports

## Supervisor Quick Start

### Your Daily Workflow

```
1. HUB OVERVIEW       → Check hub operations dashboard
2. APPROVAL QUEUE     → Process withdrawal requests
3. TEAM MANAGEMENT    → Monitor field team status
4. ISSUE RESOLUTION   → Handle escalations
5. REPORTING          → Generate hub reports
```

### Key Pages for You

| Task | Where to Go |
|------|-------------|
| View hub operations | Hub Operations |
| Approve withdrawals | Withdrawal Approval |
| Manage team | Field Team |
| View classifications | Classifications |
| Generate reports | Reports |
| Monitor sites | Site Visits |

### Dashboard Filtering

Your dashboard shows data filtered by your assigned Hub:
- All metrics reflect only your hub's data
- Team members shown are in your hub
- Financial summaries are hub-specific

### Withdrawal Approval Process

As a supervisor, you provide **first-tier approval**:

1. Go to **Withdrawal Approval**
2. Review pending requests
3. For each request, verify:
   - Requester has sufficient balance
   - Amount is reasonable
   - No suspicious patterns
4. Click **Approve** (moves to Finance)
   OR **Reject** (provide reason)

**Approval Flow:**
```
User Request → YOU (Supervisor) → Finance/Admin → Paid
```

### Hub Operations Management

1. Go to **Hub Operations**
2. View:
   - Sites in your hub
   - Team performance
   - Completion rates
   - Pending tasks
3. Actions:
   - Assign sites to team
   - Reallocate resources
   - Generate hub reports

### Managing Team Performance

1. Review team workload
2. Check completion rates
3. Identify bottlenecks
4. Support struggling team members
5. Recognize high performers

### Tips for Success

- Process approval queue daily
- Balance team workloads
- Document rejection reasons clearly
- Escalate financial anomalies
- Generate weekly hub summaries

## Finance Officer Quick Start

### Your Daily Workflow

```
1. APPROVAL QUEUE     → Process finance approvals
2. WALLET REVIEW      → Monitor user wallets
3. BUDGET CHECK       → Track budget utilization
4. PAYMENT PROCESSING → Complete payouts
5. REPORTING          → Generate financial reports
```

### Key Pages for You

| Task | Where to Go |
|------|-------------|
| Final approvals | Finance Approval |
| View all wallets | Admin → Wallets |
| Manage budgets | Budget |
| Financial overview | Financial Operations |
| Generate reports | Wallet Reports |
| View transactions | Finance |

### Finance Approval Process

You provide **final approval** for withdrawals:

1. Go to **Finance Approval**
2. Review supervisor-approved requests
3. For each request, verify:
   - Bank details are correct
   - Amount matches request
   - No duplicate requests
4. Click **Approve & Process**
   OR **Reject** (provide reason)

**Approval Flow:**
```
User Request → Supervisor Approved → YOU (Finance) → Paid
```

### Processing Payouts

After approval:
1. Verify bank account details
2. Initiate transfer
3. Mark as "Processing"
4. When confirmed, mark as "Paid"
5. User wallet updated automatically

### Budget Management

1. Go to **Budget** page
2. Monitor:
   - Total allocated
   - Amount spent
   - Remaining balance
   - Burn rate
3. Set alerts for:
   - 75% utilization
   - 90% utilization
   - Over budget

### Wallet Oversight

1. Go to **Admin → Wallets**
2. View all user wallets
3. Check for:
   - Unusual balances
   - Suspicious transactions
   - Processing delays
4. Click on user for detailed view

### Tips for Success

- Process approvals same day if possible
- Document all rejection reasons
- Monitor budget burn rates
- Flag suspicious activities
- Reconcile daily

## Field Operation Manager (FOM) Quick Start

### Your Daily Workflow

```
1. MMP REVIEW         → Review pending MMPs
2. DISPATCH           → Assign coordinators
3. PROGRESS TRACKING  → Monitor field progress
4. COVERAGE ANALYSIS  → Check tracker plan
5. ISSUE MANAGEMENT   → Handle escalations
```

### Key Pages for You

| Task | Where to Go |
|------|-------------|
| Review MMPs | MMP |
| Assign coordinators | MMP → Review & Assign Coordinators |
| Track progress | Tracker Preparation Plan |
| Monitor field ops | Field Operation Manager |
| View sites | Site Visits |
| Team overview | Field Team |

### MMP Workflow

1. **Receive** - MMP uploaded and forwarded to you
2. **Review** - Verify site data quality
3. **Assign** - Assign coordinators to sites
4. **Dispatch** - Send to field team
5. **Monitor** - Track completion progress
6. **Report** - Generate coverage reports

### Assigning Coordinators

1. Open an approved MMP
2. Click **Review & Assign Coordinators**
3. View sites grouped by state/locality
4. For each area:
   - See available coordinators
   - Check their current workload
   - Select appropriate coordinator
5. Review all assignments
6. Click **Confirm & Dispatch**

### Tracker Preparation Plan

Use for coverage analysis:
1. Go to **Tracker Preparation Plan**
2. Select project/MMP
3. View:
   - Planned vs. actual sites
   - Coverage gaps
   - Completion timeline
4. Export for reporting

### Site Dispatch Modes

When dispatching, choose mode:

| Mode | Who Can Claim |
|------|---------------|
| Open | Any enumerator |
| State | Enumerators in that state |
| Locality | Enumerators in that locality |
| Individual | Specifically assigned user |

### Tips for Success

- Review MMPs within 24 hours
- Balance coordinator workloads
- Monitor coverage daily
- Address gaps proactively
- Prepare weekly progress reports

## Reviewer Quick Start

### Your Role

Reviewers have read-only access to verify and review data across the platform.

### Your Access

```
READ-ONLY ACCESS to:
- Dashboard metrics
- MMP data
- Site visit information
- Reports and analytics
- Project information
```

### Key Pages for You

| Task | Where to Go |
|------|-------------|
| View dashboard | Dashboard |
| Review MMPs | MMP |
| Check site visits | Site Visits |
| View projects | Projects |
| Access reports | Reports |

### What You CAN Do

- View all MMPs and their sites
- Review site visit data
- Access dashboard metrics
- Generate and export reports
- View project information
- Check user activity (read-only)

### What You CANNOT Do

- Create or edit MMPs
- Modify site visits
- Approve withdrawals
- Change user accounts
- Edit project data
- Make financial transactions

### Review Workflow

1. **Access Data**: Navigate to relevant section
2. **Review**: Examine data for accuracy
3. **Note Issues**: Document any problems found
4. **Report**: Submit findings to appropriate team
5. **Follow Up**: Track resolution

### Tips for Success

- Focus on data quality
- Document all findings
- Communicate issues clearly
- Use filters to narrow focus
- Export data for detailed analysis
